# 改进日志：记忆系统 v4.0 — 自动捕获 + 通用桥接API

**日期**: 2026-05-26
**背景**: 对比 agentmemory 的六生命周期Hook机制，发现伯仕记忆系统在自动捕获、三路检索、预取和跨Agent共享四个维度存在缺口。
**工具链**: Python 3.11, ChromaDB, Sentence-Transformers, KnowledgeGraph, HTTP.server

---

## 改造内容

### 1. 自动捕获引擎 (`auto_capture.py`)
- 对标 agentmemory 的 sync_turn / prefetch / on_session_end / system_prompt_block
- 六合一脚本：capture / prefetch / search / brief / session-end / status
- 三路检索：BM25近似(关键词) + 向量语义 + 知识图谱
- cron每30分钟自动执行

### 2. 通用记忆桥接API (`memory_bridge_api.py`)
- HTTP 服务 on port 3112
- 5个端点：GET /health, GET /memory/status, POST /memory/search, POST /memory/save, POST /memory/brief
- 任意Agent（小乐、OpenCode、Claude Code等）可通过HTTP读写伯仕记忆
- 零依赖，纯Python标准库

### 3. 跨Agent共享桥接
- 共享目录：`~/.openclaw/shared/boshi_memory_bridge/`
- 快照文件：`latest_capture.json`（每30分钟更新）
- 接入指南（含OpenCode MCP配置示例、小乐Python示例）

### 4. Persona规则更新
- 铁律④：每次对话结束前自动boshi_conclude存关键信息
- 铁律⑤：每次对话开始先三路检索相关记忆
