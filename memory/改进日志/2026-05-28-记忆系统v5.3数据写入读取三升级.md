# 改进日志：伯仕记忆系统 v5.3 — 数据层+写入层+读取层三升级

**日期**: 2026-05-28
**背景**: 乐之指出新会话接不上旧话题、提旧事不知所云、记忆无标签关联。经过推敲讨论，设计了三联标签+场景化存储+全量清洗+新会话三连扫方案，并在同一天全部实现。
**工具链**: DeepSeek V4 Flash (API), Hermes delegate_task (并行子代理), Python 3.11, hot.json, knowledge_graph.json, boshi_conclude, json.dumps

---

## 改造内容

### 1. 数据层升级 — 全量记忆标签清洗 (v5.3)

- **hot.json**: 1,049 个话题全部补上 `category`(大类::小类)、`entities`(关联实体)、`status`(活跃/已放弃/待定/测试/普通)、`description`(简短描述) 四个字段
- **knowledge_graph.json**: 136 个节点全部补上同样四字段
- 分类覆盖：40+ 子类，从技术::工具使用到文学::古籍到元::记忆设计
- 状态分布：普通 541 / 活跃 429 / 测试 49 / 待定 25 / 已放弃 5
- 费用：~$0.52（297万token，DeepSeek V4 Flash分批处理）
- 处理方式：hot.json分5批并行（delegate_task）+ 知识图谱1批，每批模型自动分类

### 2. 写入层升级 — boshi_conclude 强制结构化 (v5.1)

- 所有新记忆写入必须采用 JSON 格式编码到 content 字段：
```json
{
  "type": "场景/事实/决策",
  "fact": "核心事实",
  "class": "大类::小类",
  "entities": ["实体1", "实体2"],
  "scene": "所属场景",
  "heat": 0-10,
  "status": "活跃/已放弃/待定",
  "anchor": "上下文锚点"
}
```
- heat 规则：正在讨论 7-9，日常参考 3-6，冷却 1-2，已放弃 0
- topic 字段填最相关的大类名
- 查询时先尝试 JSON 解析，失败走传统语义搜索

### 3. 读取层升级 — 新会话三连扫 + 模糊词自动扩展 (v5.2)

- **新会话三连扫**：开新会话自动执行：
  1. `boshi_search("焦点话题", top_k=3)` — 按热度排前
  2. 查最近 24 小时活跃话题
  3. 查 status=待定 的事项
- **模糊词自动扩展**：用户说"那个事""上次那个"等模糊词时：
  1. 先匹配热区焦点话题（当前活跃范围）
  2. 匹配不上再到所有记忆中做标签扩展
  3. 两轮都匹配不上再问用户
- 行为规则已固化到 memory（【写入层v2.2】+【读取层v2.2】条目）

## 审查发现

### 解决的核心痛点
1. **开新会话不知道最近在忙什么** → 三连扫自动报出热区焦点
2. **提旧事不知所云** → 标签扩展+实体关联，从话题名就能定位场景
3. **已放弃话题反复问** → 状态标记 "已放弃"，heat=0 不再进入焦点
4. **记忆碎片无关联** → 同一标签集群自然关联所有相关记忆

### 待解决的问题
1. **cross_platform_history.json**（95KB 跨平台会话摘要）尚未补标签 — 留待下次巡检
2. **348 条超低热度冷话题**（heat<0.01，测试消息/死对话线）虽然已有分类但实际不会进入焦点 — 可归档冷区
3. **三连扫的自动化程度** — 依赖行为规则自觉执行，不是代码级自动触发
4. **标签一致性机制** — 目前靠我手动保持，后续需建立自动标签规范
