# 伯仕记忆系统 v4.0 — 统一存储 + 热度调度

> 最后更新：2026-05-26
> 系统版本：v4.0（统一 Chroma + tier 热度标签）
> 存储：ChromaDB（统一）+ state.db（兜底）

---

## 一、核心设计理念

**四层不是四个数据库，是四套读取策略。** 数据统一存在 Chroma 中，用 tier/heat 字段控制调度。

| 区 | 读取规则 | Chroma 标签 |
|:--|:---------|:-----------|
| 🔥 热区 | 每轮对话自动加载到上下文 | heat ≥ 10 |
| 🌡️ 温区 | boshi_search 语义搜索命中 | 1 ≤ heat < 10 |
| ❄️ 冷区 | boshi_excavate 深度挖掘 | heat < 1 |
| 🕉️ 全量 | session_search 查原始记录 + state.db 兜底 | raw_conversation 类型 |

---

## 二、存储层（统一 Chroma）

所有记忆统一写入 ChromaDB，不再使用 hot.json / cold.json 等独立文件：

| 类型 | 内容 | 初始 heat | 用途 |
|:-----|:-----|:---------:|:-----|
| raw_conversation | 完整原始对话 | 15 | 全量记录 |
| conversation_turn | 对话摘要（各200字） | 20 | 快速回顾 |
| work_log | 工作日志 | 25 | 开发追踪 |
| project_log | 项目日志 | 30 | 项目溯源 |
| knowledge | 知识/联想 | 20 | 学习积累 |
| fact / decision 等 | 关键事实 | 15 | 语义搜索 |

---

## 三、写入流程

```
每轮对话结束 → sync_turn()
    ├→ (1) 写对话摘要 → Chroma (type:conversation_turn, heat:20)
    ├→ (2) 写完整原文 → Chroma (type:raw_conversation, heat:15)
    ├→ (3) 热度更新 → 检测已有话题，匹配则 heat += 8
    ├→ (4) 写关键事实 → Chroma (type:fact/decision 等)
    └→ (5) 自动工作日志 → Chroma (type:work_log)
```

所有写入同步进行，不阻塞对话。

---

## 四、读取流程（热度调度）

### 4.1 prefetch（每轮对话启动时）

```
新对话 → queue_prefetch()
    ├→ (1) 查 Chroma heat ≥ 10 → top 15 → 热区话题 → 前10条注入上下文
    ├→ (2) 查 state.db 最近2天 → 跨通道对话摘要
    ├→ (3) 检测项目关键词 → 加载 project_log
    └→ (4) 语义搜索 → Chroma query + 冷区兜底
```

### 4.2 boshi_search

```
搜索 → Chroma 语义搜索
    结果按 heat DESC 排序
    先显示高热度话题
```

### 4.3 热度衰减（每次对话启动时自动触发）

```
每次新对话 → queue_prefetch()
    ↓
检查 .last_heat_decay 标记
    ├─ 距上次 < 24小时 → 跳过
    └─ 距上次 ≥ 24小时 → 执行
        ↓
    Chroma 所有条目 heat *= 0.9
    heat ≥ 10 → tier = hot
    1 ≤ heat < 10 → tier = warm
    heat < 1 → tier = cold
    记录本次时间到标记文件
```

不依赖 cron，不依赖设备在线时间。

## 五、与 v3.2 的变更对比

| 维度 | v3.2 | v4.0 |
|:-----|:-----|:------|
| 热区 | hot.json 独立文件 | Chroma heat ≥ 10 |
| 温区 | Chroma 本体 | Chroma 1 ≤ heat < 10 |
| 冷区 | cold.json 独立文件 | Chroma heat < 1 |
| 全量原始对话 | 仅 state.db | Chroma + state.db 双保险 |
| 热度管理 | hot.json 内置衰减 | Chroma metadata heat 字段 |
| 跨库一致性 | ❌ 可能不同步 | ✅ 单库无同步问题 |
| 查询一套代码 | ❌ 需查 3 个来源 | ✅ 全走 Chroma |

## 六、数据流全览

```
              写入端                         读取端
  ┌────────────────────┐       ┌──────────────────────┐
  │ CLI 对话            │       │ 新对话 → prefetch()   │
  │ 微信对话            │       │   ↓                  │
  │ 工作台对话          │       │  Chroma heat≥10 →    │
  │ WebUI 对话          │       │  上下文              │
  │   ↓ sync_turn()    │       │                      │
  │ 全部写入 Chroma    │       │ 搜索 → boshi_search   │
  │ （摘要+原文+热度）  │       │   ↓                  │
  └────────────────────┘       │  Chroma 语义排序     │
                               │  + heat 降序         │
                               └──────────────────────┘

  ┌──────────────────────────────────────────────┐
  │  ChromaDB（统一存储）                          │
  │  raw_conversation / conversation_turn         │
  │  work_log / project_log / knowledge           │
  │  每条带 heat + tier + last_mentioned          │
  └──────────────────────────────────────────────┘
            ↕
  ┌──────────────────────────────────────────────┐
  │  state.db（Hermes 原生全量，兜底不动）          │
  └──────────────────────────────────────────────┘
```
