## 三、ChromaDB Schema

### 3.1 通用 metadata 字段

| 字段 | 类型 | 说明 |
|:-----|:-----|:-----|
| `type` | string | 记录类型（conversation_turn / fact / relation / profile_static 等） |
| `isLatest` | bool | 是否为当前有效版本（版本链核心字段） |
| `tier` | string | hot / warm / cold |
| `heat` | float | 热度值（衰减规则：每日 ×0.9，heat<10 变 warm，heat<1 变 cold） |
| `source` | string | 数据来源 |
| `created_at` | float | 创建时间（Unix 时间戳） |
| `collected_at` | float | 采集时间 |

### 3.2 版本链专用字段

| 字段 | 说明 |
|:-----|:-----|
| `isLatest` | bool，当前有效版本 |
| `_parent_id` | 被更新的旧版本 ID |
| `_superseded_by` | 替代此版本的新版本 ID |
| `_version_created` | 版本创建时间 |
| `_deprecated_at` | 折旧时间 |
| `isForgotten` | 是否已被自动遗忘 |
| `forget_after` | 过期时间戳 |
| `forget_reason` | 遗忘原因 |

### 3.3 记录类型一览

| type | 数量 | 用途 |
|:-----|:--:|:-----|

## 七、降级防护

CN Desktop 污染导致 ChromaDB 的 `where` 过滤在部分数据上崩溃。v6.0 内置两层防护：

```python
# search_memory 降级
try:
    results = col.query(where={"type": "fact", "isLatest": True})
except InternalError:
    # 完全降级：无 where 大查询 + Python 端 isLatest+type 过滤
    results = col.query(n_results=50)
    # 手动过滤 metadata

# auto_forget 降级
try:
    results = col.get(limit=500, offset=offset)
except InternalError:
    # 跳过损坏批次，继续下一批
    offset += batch_size
    continue
```

2026-06-11 已修复 ChromaDB collection（重建 11,184 条），降级路径目前不触发，但保留作为保险。

## 八、部署

### 8.1 环境要求

- Python 3.11+
- ChromaDB (`pip install chromadb`)
- SentenceTransformer (`pip install sentence-transformers`)
- Hermes Agent (MemoryProvider 插件框架)

### 8.2 启动流程

```
1. Hermes 启动 → 加载 boshi MemoryProvider
2. initialize() → 配置自检 (memory.provider=boshi)
3. system_prompt_block() → 注入版本信息 + 项目索引 + 资源索引
4. queue_prefetch() → 后台加载热区 + 最近 2 天跨通道摘要
5. sync_turn() → 每轮对话自动采集+画像更新+图谱关联
```

## 九、开放接口（v6.1 新增）

### 9.1 设计原理

MCP Server + CLI 双轨开放接口，让6.0记忆系统不再只被Hermes插件独占：

- **MCP Server** → 给外部Agent用（Claude Code、Cursor、Hermes等支持MCP协议的客户端）
- **CLI** → 给人和脚本用（终端直接操作，可组合进自动化流水线）
- **boshi_core.py** → 共享底层，两者都通过它访问chroma_bridge和knowledge_graph，不重复实现业务逻辑

### 9.2 架构图

```
chroma_bridge.py (6.0原有，不动)
knowledge_graph.py (5.0原有，不动)
        ↗
boshi_core.py (v6.1新增，只调不改)
     ↗           ↗
boshi_mcp_server.py   boshi_cli.py
  (MCP Server)          (CLI)
```

### 9.3 MCP Server Tools

| Tool | 对应Core函数 | 说明 |
|:-----|:-----|:-----|
| `boshi_search` | search() | 多策略检索（all/vector/hybrid/graph） |
| `boshi_save` | save() | 存入记忆 |
| `boshi_delete` | delete() | 删除记忆 |
| `boshi_status` | status() | 记忆库状态 |
| `boshi_profile` | profile() | 用户画像 |
| `boshi_graph` | graph_query() | 知识图谱BFS查询 |
| `boshi_graph_add` | graph_add_node/edge() | 添加图谱节点/边 |
| `boshi_recent` | recent() | 最近N条记忆 |

### 9.4 CLI 子命令

```
boshi search "查询" [--top-k 5] [--source all|vector|hybrid|graph]
boshi save "内容" [--topic 主题]
boshi delete <id>
boshi status
boshi profile
boshi graph <实体> [--depth 2]
boshi graph-add-node <名称> [--type 类型]
boshi graph-add-edge <起点> <终点> <关系>
boshi recent [10]
```

### 9.5 Hermes 连接配置

```yaml
# config.yaml
mcp_servers:
  boshi:
    command: "python"
    args: ["~/.boshi/boshi_mcp_server.py"]
```

### 9.6 技术要点

- **惰性导入**：boshi_core.py首次调用时才加载chroma_bridge（含embedding模型），避免启动卡顿
- **MCP SDK版本**：需 mcp ≥ 1.27，装饰器必须加括号：`@server.list_tools()` / `@server.call_tool()`
- **三路检索映射**：source="all" → hybrid_search+graph_search融合，source="vector" → search_memory，source="hybrid" → hybrid_search，source="graph" → graph_search
- **不改动6.0任何原有代码**：三个新文件全是外层壳，chroma_bridge/knowledge_graph/插件__init__.py一行未动
