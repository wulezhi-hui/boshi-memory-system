## 一、核心设计理念

**物理统一·逻辑分层 — 2026-06-15 乐之确立**

记忆数据必须统一在一个数据库。热区/温区/冷区是用标签+时间戳来区分，是记忆提取策略的问题，不应该是物理上的隔离。

| 区 | 读取规则 | Chroma 标签 |
|:--|:---------|:-----------|
| 🔥 热区 | 每轮对话自动加载到上下文 | heat ≥ 10 |
| 🌡️ 温区 | boshi_search 语义搜索命中 | 1 ≤ heat < 10 |
| ❄️ 冷区 | boshi_excavate 深度挖掘 | heat < 1 |
| 🕉️ 全量 | session_search + state.db | raw_conversation 类型 |

**拟人化比喻：**
- 🔥 桌面（热区）— 正在忙的几个项目一直摊在桌上，不会因为去泡茶就被收走
- 📚 书架（温区）— 做完的项目放书架上，知道有什么，需要时走过去翻
- 🏚️ 阁楼（冷区）— 久远的事平时不想着，但一提关键线索就想起全貌

## 三、向量引擎

| 项 | 值 |
|:---|:----|
| 模型 | sentence-transformers/all-MiniLM-L6-v2 |
| 维度 | 384 |
| 大小 | ~90MB |
| 外网依赖 | 零（已缓存到本地 HuggingFace cache） |
| 模型路径 | `~/.cache/huggingface/.../snapshots/c9745ed1d9f...` |
| 代码入口 | `~/.boshi/chroma_bridge.py` |
| 嵌入函数 | `SentenceTransformerEmbeddingFunction`（单例缓存） |
| 引擎历史 | v4 BGE-M3(1024维,Ollama依赖) → v5.0 ONNX(外网依赖) → **v5.3+ all-MiniLM-L6-v2(零外网)** |

### 重要注意事项
1. 不要用 `ONNXMiniLM_L6_V2` — 它会强制联网下载，即使本地已缓存
2. Embedding 函数需做单例缓存，否则每次调用重加载模型（5-10秒）
3. 修改代码后必须重启 Hermes（Python 模块缓存问题）

## 四、Hermes 框架集成

### 4.1 配置（config.yaml）

```yaml
memory:
  memory_enabled: false          # 关闭内置 MEMORY.md
  user_profile_enabled: false    # 关闭内置 USER.md
  provider: boshi                # 仅 boshi 负责所有记忆
  flush_min_turns: 6
```

### 4.2 三层自愈架构

| 层级 | 保护对象 | 机制 |
|:----|:---------|:-----|
| L1 | config 被重置 | `initialize()` 自动 `hermes config set memory.provider boshi` |
| L2 | plugin 文件被删 | 用户级路径 `$HERMES_HOME/plugins/boshi/` 不被升级清理 |
| L3 | 数据目录丢失 | `initialize()` 检测 chroma_db 缺失，自动调 deploy.py 恢复 |

### 4.3 Boshi Plugin

位置：`$HERMES_HOME/plugins/boshi/__init__.py`
功能：`prefetch()` 对话上下文注入 + `sync_turn()` 实时对话存储
