## 2026-06-18 — v6.1 向量层重构（ONNX 替代 SentenceTransformer，零外部依赖）

**操作：** upgrade
**文件：** chroma_bridge.py、onnx_embed.py、install.sh、install.sh
**详情：**
- Embedding 引擎从 `SentenceTransformer(paraphrase-multilingual-MiniLM-L12-v2)` 重构为 ChromaDB 内置 `ONNXMiniLM_L6_V2`
  - 不再依赖：`torch` (~1.5GB)、`transformers`、`sentence-transformers`、`HuggingFace` 缓存
  - 保留依赖：仅需 `chromadb`（已内含 `onnxruntime` + `tokenizers`）
  - 向量模型：`all-MiniLM-L6-v2`（384 维，ONNX 格式）
  - 仓库自带版本：`models/all-MiniLM-L6-v2/onnx/`（model.onnx + tokenizer + vocab，~87MB）
  - 加载策略：优先用仓库自带 → 缺失时 fallback 到 ChromaDB 自动下载（需联网）
- 新增 `onnx_embed.py`：封装 `BoshiEmbeddingFunction`，处理模型路径探测和 monkey-patch
- 修改 `chroma_bridge.py`：
  - 删除：`SentenceTransformerEmbeddingFunction`、`LOCAL_MODEL_PATH`（HuggingFace 缓存路径）
  - 替换：`from onnx_embed import get_embedding_function as _get_embedding_function`
- 修改 `install.sh`：
  - Ubuntu 一键安装脚本改为方案 B：仓库不打包 86MB ONNX 模型
  - 安装时自动触发 ChromaDB 内置下载机制（首次实例化 `ONNXMiniLM_L6_V2`）
  - 或从 `~/.cache/chroma/onnx_models/` 复制到 `~/.boshi/models/`
- GitHub 仓库推送：
  - 核心代码：onnx_embed.py、chroma_bridge.py、install.sh
  - 仓库体积：不含 model.onnx，用户 clone 后即拥有代码（模型按需下载）
- 兼容性：向量维度仍为 384，现有 ChromaDB collection 数据无损，无需重建
- 历史脉络：
  - v4.0：BGE-M3 (1024维, Ollama 依赖) → v5.0：ONNX (外网下载) → v5.3：all-MiniLM-L6-v2 (HuggingFace 缓存) → **v6.1：ONNX 自带/自动下载 (零重型依赖)**
