# 伯仕记忆读取优化方案 v2.1

> 目标：新会话打开无缝接旧话题，一提关键词整个场景浮现。

---

## 一、核心模型：场景化记忆

不再以「话题」为记忆单元，而以「场景」为单位。

### 场景 = 时间窗口 + 话题集群 + 参与者 + 决策 + 上下文

```
场景示例：
  时间：2026-05-25 晚
  话题：ds-free-api / Camofox / 502 / API代理
  参与者：乐之 → 决定放弃
  上下文：尝试多种修复方案后，乐之最终从模型列表删除
  热度：8 → 因用户说"放弃了"而降至 0
```

---

## 二、记忆存储规范

### 2.1 存储格式

每条记忆采用结构化 content，包含：

```
[事实] ds-free-api 启动在 23333 端口返回 502 错误
[大类] 技术::调试
[实体] ds-free-api | Camofox | 502错误
[关系] ds-free-api → 上游: Camofox | 状态: 异常
[场景] 2026-05-25晚 ds-free-api调试会话
[热度] 7
[状态] 活跃
[锚点] 用户尝试让OpenClaw通过网页版API省费用，Camofox在9377端口运行正常
```

### 2.2 标签规范

**大类（6类，不增不减）：**

| 大类 | 示例 | 说明 |
|------|------|------|
| 技术 | 调试、配置、工具使用、API、代码 | 技术相关的一切 |
| 文学 | 古籍、佛经、写作、校对 | 文化内容 |
| 工作方式 | 行为规则、沟通风格、协作流程 | 怎么做事 |
| 项目 | 按项目名 | 虚拟寺院、书库整理等 |
| 决策 | 放弃、采用、待定、暂停 | 结论类 |
| 元 | 记忆设计、进化机制 | 关于系统本身的讨论 |

**实体标签规范：**
- 工具名保持原名（ds-free-api, Camofox, Clash）
- 人名固定（乐之、小乐、伯仕）
- 状态标签统一（活跃/已放弃/待定/暂停）

**标签一致性机制：**
每次打标签前做三步：
1. 查已有标签库中是否有同一实体的标签
2. 如果有，复用已有标签名
3. 如果新增，统一命名风格（首字母小写、连字符分隔）

### 2.3 热度管理

| 因素 | 权重 | 说明 |
|------|------|------|
| 最近活跃时间 | 高 | 越近越热 |
| 提及频率 | 中高 | 连续3次以上提到同一场景，热度+1 |
| 用户态度 | 极高 | "不用了"="已放弃"→热度归零 |
| 时间衰减 | 每日-1 | 7天没提自动降级到温区，30天到冷区 |

热度值范围：0-10。大于等于7在🔥热区，4-6在🌡️温区，0-3在❄️冷区。

## 三、读取策略

### 3.1 新会话自动「三连扫」

```
═══════════════════════════════════════

[🔥] 记忆读取优化方案设计 — 正在讨论，heat=9
[🔥] 虚拟寺院内容分发 — 推进中，heat=7

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 最近活跃（24小时内）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```

### 3.2 用户提词 → 三层扩展查询

```
第一层：标签精确匹配
  "代理" → 标签库中有多少场景含"代理"
  
第二层：同义词扩展 
  "那个网页版" → 无精确匹配
  → 同义词扩展："网页版" ≈ "web" ≈ "网页" ≈ "Camofox"
  → Camofox → ds-free-api 场景

第三层：关联扩展
  ds-free-api 场景 →
    ├── Camofox（浏览器自动化）
    ├── 502错误（症状）
    └── 已放弃（最终状态）
```

### 3.3 每次查询后自动补关联

查到相关场景后，自动记录这次查询：
```
[关联] 用户新会话提到"代理" → 关联到 ds-free-api 场景
```
这样下次再查"代理"时，关联更强。

## 四、话题生命周期

### 自动升温曲线

```
第1天：提了1次 → heat 3 （刚进温区）
第2天：又提了3次 → heat 6
第3天：集中讨论 → heat 9 （🔥热区）
5天后没再提 → heat 4 （🌡️温区）
15天后没提 → heat 1 （❄️冷区）
```

### 用户触发关闭

你说"不用了/删掉了/放弃了/不要了"
→ 我立即写一条决策记忆，标记该话题 `状态=已放弃`，热度归零
→ 新会话的🔥焦点不再显示

## 五、实现路径

| 阶段 | 内容 | 交付物 |
|------|------|--------|
| **Phase 1 ✅** | 新记忆写入规范 | 从今天起每条新记忆带标签+场景+热度+状态 |
| **Phase 2** | 新会话三连扫 | 每次开新会话自动输出焦点/最近/待定 |
| **Phase 3** | 用户提词三层扩展 | 用户提一个词，自动做标签-同义-关联三层展开 |
| **Phase 4** | 存量记忆清洗 | 按优先级回填旧记忆的标签和场景 |

### Phase 1 落地方式

采用轻量方式——不改工具代码，通过结构化 content 实现：

```python
boshi_conclude(
    content=json.dumps({
        "type": "场景记忆",
        "fact": "ds-free-api 已放弃，乐之从模型列表删除",
        "class": "决策::放弃",
        "entities": ["ds-free-api"],
        "relations": [("ds-free-api", "最终决策", "已放弃")],
        "scene": "2026-05-25晚 ds-free-api调试",
        "heat": 0,
        "status": "已放弃"
    }, ensure_ascii=False),
    topic="ds-free-api"
)
```

查询时先解析 JSON，能解析的按结构化字段查询，不能解析的（旧记忆）走传统语义搜索。
