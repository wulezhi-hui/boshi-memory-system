## 一、核心模型：场景化记忆

不再以「话题」为记忆单元，而以「场景」为单位。

### 场景 = 时间窗口 + 话题集群 + 参与者 + 决策 + 上下文

```
场景示例：
  时间：2026-05-25 晚
  话题：ds-free-api / Camofox / 502 / API代理
  参与者：乐之 → 决定放弃
  上下文：尝试多种修复方案后，乐之最终从模型列表删除
  热度：8 → 因用户说"放弃了"而降至 0
```

## 二、记忆存储规范 (v5.1 写入层)

### 2.1 存储格式

每条记忆 content 必须采用 JSON 格式：

```json
{
  "type": "场景/事实/决策",
  "fact": "核心事实",
  "class": "大类::小类",
  "entities": ["实体1", "实体2"],
  "scene": "所属场景",
  "heat": 0,
  "status": "已放弃",
  "anchor": "上下文锚点"
}
```

查询时先尝试 JSON 解析（新记忆），失败走传统语义搜索（旧记忆）。

### 2.2 标签规范

**六大类（25+ 小类）：**

| 大类 | 小类示例 | 说明 |
|------|----------|------|
| 技术 | 工具使用/配置/调试/API/代码/性能/架构/文档 | 技术相关 |
| 文学 | 古籍/佛经/写作/校对/文化 | 文化内容 |
| 工作方式 | 沟通/行为规则/协作/偏好 | 怎么做事 |
| 项目 | 虚拟寺院/书库整理/内容分发/工作台/UE5/记忆系统 | 按项目 |
| 决策 | 放弃/采用/待定/暂停 | 结论类 |
| 元 | 记忆设计/系统/进化 | 系统本身 |

**热度规则：**
| 范围 | 含义 | 说明 |
|------|------|------|
| 7-10 | 🔥 热区 | 正在讨论中 |
| 3-6 | 🌡️ 温区 | 日常参考 |
| 1-2 | ❄️ 冷区 | 很少提及 |
| 0 | 🗑️ 已放弃 | 用户说"不要了" |

## 三、读取策略 (v5.2 读取层)

### 3.1 新会话自动「三连扫」

每次开新会话自动执行：

```
① boshi_search("焦点话题", top_k=3) → 按热度排前
② 查最近 24 小时活跃话题
③ 查 status=待定 的事项
```

输出格式：

```
═══════════════════════════════════════
```

### 3.2 用户提词 → 三层扩展查询

用户说模糊词时自动执行：

```
第一层：标签精确匹配
  说"代理" → 标签库中匹配 project::内容分发/技术::API/实体::Clash

第二层：实体+标签扩展
  说"那个网页版" → 无精确匹配
  → 标签扩展："网页版" ≈ "web" ≈ "Camofox"
  → Camofox → ds-free-api 场景 → 状态=已放弃

第三层：关联扩展
  ds-free-api →
    ├── Camofox（浏览器自动化）
    ├── 502错误（症状）
    └── 已放弃（最终决策）
```

前两轮匹配不上才问用户。

## 四、话题生命周期

### 自动升温/降温

```
第1天：提了1次 → heat 3（温区底部）
第2天：提了3次 → heat 6
第3天：集中讨论 → heat 9（🔥热区）
5天后没再提 → heat 4（🌡️温区）
15天后没提 → heat 1（❄️冷区）
```

### 用户触发关闭

你说"不用了/删掉了/放弃了"等 → 立即写 `status=已放弃, heat=0`，不再进入焦点。
